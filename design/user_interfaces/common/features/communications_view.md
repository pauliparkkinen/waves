# Description

User is able to communicate with other users effectively. Usage of written text is disencouraged especially on the HCP side. The previous communications are grouped by thread and the user and selectable and reviewable later on.

# Requirements

1. User is able to select a purpose of a 'communication' from a predefined list
 - The purpose list is ordered based on relevancy calculated from the context
2. User is able to select who the 'communication' is sent to.
 - The communication should be limited to one hospital (group) only.
3. User can attach free text if needed.
 - Templates should be configurable in the admin view for free text for different purposes
4. The communication can be tied to one or more contexts, for instance an answer to a questionnaire, status or something else.
 - There should be a connection between a 'communication' and an anwer to it.
 - Communication without context should be disencouraged
5. User is able to trigger a communication from a context
6. User is able to forward communication to another user (that has access to the contexts involved)
7. The communication purpose is preselected based on the context. For instance an answer to a certain type of 
8. User can attach documents if needed
9. The communications view should be a popup so that the current context can be reviewed.
